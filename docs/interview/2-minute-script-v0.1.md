# 2-Minute Interview Script (v0.1)

## When to use
- Recruiter screen
- Quick self-introduction in technical interview
- Networking conversation

## Core script (general)
I’m building an end-to-end project called the Public Renewable Grid Resilience Auditor.

The business problem is that public-sector energy planners need reliable and explainable indicators for renewable integration, grid stress, and policy risk, but data is fragmented across sources.

My solution is a contract-first analytics platform using free public APIs like ENTSO-E, Open-Meteo, and Ember. I use a Bronze/Silver/Gold architecture with DuckDB and Parquet, then model policy KPIs and forecasting outputs for decision support.

So far, I’ve completed governance and KPI definition, environment and CI standards, source contracts with explicit quality rules, and raw ingestion jobs with retry/backoff and idempotent metadata-aware writes.

What I want interviewers to see is not only that I can write code, but that I can design reliable analytics systems: clear contracts, reproducible workflows, quality gates, and stakeholder-ready outputs.

I’m continuing toward staging models, marts, forecasting, dashboarding, and orchestration, so this project demonstrates both implementation depth and product thinking.

## Role-focused endings (pick one)

### Data Analyst ending
I focus on translating technical pipelines into understandable KPI narratives so decision-makers can act with confidence.

### Analytics Engineer ending
I focus on semantic consistency, tested transformations, and governance-driven analytics delivery.

### Data Engineer ending
I focus on resilient ingestion, idempotent processing, and operational reliability under evolving source conditions.

### Stack Analytics Engineer ending
I focus on owning the full lifecycle from source contracts through data modeling to stakeholder-facing analytics products.
